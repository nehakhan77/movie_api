Movie API

Objective: To build a server-side component of a "movies" web application. The web application will provide users to access information about a variety of movies, directors, and genres they have watched or are interested in. Users will be able to sign up, update their personal information, and create a list of their favorite movies!

Business Logic URL HTTP method Request body data format Response body data format
Return a list of ALL movies to the user /movies GET None A JSON object holding data about all movies
Return data about a single movie by title to the user /movies/[title] GET None A JSON object holding data about a single movie, containing title, year, genre, director. Example: { "title": "Elf", "year": "2003", "genre": { "genreName": "Comedy", "description": "Comedy is a genre of fiction that consists of discourses or works intended to be humorous or amusing by inducing laughter." }, "director": { "directorName": "Jon Favreau", "birth": "1966" } }
Return data about a genre by name /movies/genre/[genreName] GET None A JSON object holding data about a single genre, containing genre name, description. Example: { "genreName": "Comedy", "description": "Comedy is a genre of fiction that consists of discourses or works intended to be humorous or amusing by inducing laughter." }
Return data about a director by name /movies/directors/[directorName] GET None A JSON object holding data about a single director, containing director name, birth year. Example: { "directorName": "Stanley Kubrick", "birth": "1928" }
Allow new users to register /users POST A JSON object holding data about the user to add, structured like: { "name": "Neha", "favoriteMovies": [] } A JSON object holding data about the user that was added, including an ID. Example: { "name": "Neha", "favoriteMovies": [], "id": "20-10563145-8" }
Allow users to update their user info (username) /users/[id] PUT A JSON object holding data about the user which needs to be updated, structured like: { "name": "Neha", "Email": [neha@aol.com] } A JSON object holding data about the updated user information. Example: { "id": 2, "name": "Neha", "favoriteMovies": [] }
Allow users to add a movie to their list of favorites /users/[id]/[movieTitle] POST None Message indicating whether the user added the movie successfully.
Allow users to remove a movie from their list of favorites /users/[id]/[movieTitle] DELETE None Message indicating whether the user removed the movie successfully.
Allow existing users to deregister /users/[id] DELETE None Message indicating whether the user deregistered successfully.
